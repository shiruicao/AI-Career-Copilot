# 💼 AI Job Copilot · AI求职助手

一个基于 Dify 工作流编排 + Streamlit 前端构建的智能求职决策辅助系统，输入简历与岗位JD，自动输出结构化的匹配度分析、风险提示与可执行的职业建议。

> ⚠️ **关于本项目的说明**：本项目为个人自用工具，后端依赖私有部署的 Dify 工作流与 API Key，未公开部署、未开源后端服务，仓库仅展示前端代码与系统架构设计思路，不提供可直接运行的完整服务。

---

## 功能概览

输入一份简历文本和一份岗位JD文本，系统会自动完成以下分析：

- **多维度匹配评分**：从技能匹配、项目经验匹配、行业背景匹配、加分项匹配四个维度，给出加权综合评分与匹配等级
- **核心优势与差距识别**：基于简历与JD的逐项比对，列出候选人具备的核心优势，以及JD要求但简历未体现的差距项
- **求职雷区检测**：用户可自定义勾选个人不可接受的工作条件（如单休、大小周、频繁出差等），系统基于JD原文进行严格的事实核查，仅在JD中存在明确文字依据时才判定命中，避免主观联想式误判
- **岗位风险提示**：基于JD中的语言强度、职责密度等特征，识别潜在的工作强度、压力、岗位真实属性等隐性风险信号
- **行动改进指南**：分别提供简历优化建议、短中期行动计划、面试准备重点，建议均要求具体可执行、有明确时间窗口
- **完整报告导出**：一键下载完整的Markdown格式分析报告

## 系统架构

### 前端

- **框架**：Streamlit
- **可视化**：Plotly（雷达图展示多维度匹配分数）
- **设计风格**：玻璃拟态（Glassmorphism）+ 紫色系视觉语言，自定义CSS覆盖默认组件样式

### 后端：Dify 工作流编排

整个分析流程由 Dify 编排多个独立的LLM节点，按职责严格拆分，而非用单一节点承担所有任务：

```
                    ┌──→ 简历分析节点（结构化提取教育/经历/项目/技能）
                    │
开始节点（输入简历、JD、用户自定义红线条件）
                    │
                    ├──→ JD分析节点（结构化提取岗位要求/风险信号/隐性信号）
                    │         │
                    │         ↓
                    │    匹配评分节点（多维度加权评分，仅基于明确证据）
                    │         │
                    │         ↓
                    │    岗位建议节点（生成职业建议、改进行动、面试重点）
                    │         │
                    └──→ 红线核查节点（独立分支，严格事实核查，与评分逻辑完全隔离）
                              │
                              ↓
                         报告生成节点（仅做内容拼接排版，不重新分析）
                              │
                              ↓
                           输出节点
```

**关键架构设计原则**：

- **职责单一化**：每个节点只做一件事。红线核查从JD分析节点中独立拆分出来，避免"允许推断"与"禁止推断"两类相互冲突的指令出现在同一次模型调用中
- **数据单向流动**：报告生成节点不重新接触原始简历/JD数据，仅消费已经处理好的结构化结果，避免下游节点"自己再分析一次"导致内容重复或前后矛盾
- **温度控制**：评分与事实核查类节点设置 temperature = 0，保证同一输入的输出尽可能稳定一致
- **思考链分离**：针对支持 `<think>` 标签的模型，启用推理过程与结构化输出的分离，防止模型的中间推理文字污染最终的JSON结果

## 系统优势

**结构化优先于自由文本**：所有分析结果以严格的JSON Schema约束输出，前端直接读取结构化字段渲染，而非依赖正则或关键词从长文本中提取内容，避免了因措辞变化导致前端解析失败的脆弱性问题。

**事实核查与风险评估解耦**：红线检测要求"宁可漏判，不可误判"，与岗位风险评估（允许基于语言特征进行合理推断）在两个独立节点中执行，从架构层面避免了"严格事实核查"与"合理推断"两种相反逻辑相互渗透污染。

**同义词等价识别**：评分逻辑显式定义了行业术语的等价关系（如 PRD = 产品需求文档），避免模型因严格的字面匹配规则而错判明显等价的技能项。

**反幻觉约束**：所有分析节点的Prompt中均包含明确的反推测指令，例如禁止基于毕业时间推断候选人在读状态或入职时间安排，禁止将简历中"参与"、"协助"类表述升级为"主导"、"负责"。

## 技术栈

| 类别 | 技术 |
|------|------|
| 前端框架 | Streamlit |
| 数据可视化 | Plotly |
| 工作流编排 | Dify |
| LLM 模型 | 按节点任务复杂度分层选用（结构化要求高的任务使用具备强制结构化输出能力的模型；其余任务使用通义系/智谱系模型） |
| 后端通信 | Python requests（流式SSE长连接，规避网关超时） |

## 关于持续迭代

本项目在迭代过程中持续优化的方向包括：

- 提升红线检测的判定准确性，从"允许合理推断"逐步收紧为"必须有原文依据，宁可漏判"
- 解决多节点协作中因输入数据重叠而导致的内容重复生成问题
- 处理模型在长推理任务中输出格式不稳定（如Schema结构泄露进正式输出）的边界情况
- 优化前端组件的交互细节与视觉一致性
